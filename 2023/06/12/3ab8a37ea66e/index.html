查看进程线程的方法

Window系统下

  1. 任务管理器可以查看进程和线程数，也可以用来杀死进程

  2. tasklist查看进程（PID为进程编号）

  3. taskkill杀死进程（后面接**/F PID(/F表示强制结束进程)**即可）

Linux系统下

  1. ps -fe 查看所有进程
  2. ps -fT -p 查看某个进程（PID）的所有线程
  3. kill 杀死进程
  4. top 按大写H切换是否显示线程
  5. top -H -p 查看某个进程（PID）的所有线程

另外：关于Java的

  • jps 命令查看所有JAva进程
  • jstack 查看某个进程（PID）的所有线程状态
  • jconsole 来查看某个Java进程中线程的运行状况（图形界面），直接通过Win+R打开输入jconsole即可

线程运行的原理

栈与栈帧的理解、Debug

JVM是由堆、栈、方法区所组成的，而其中栈内存是给线程所使用的，每个线程启动之后，虚拟机就会为其分配一块栈内存。

  • 每个栈由多个栈帧所组成，对应着每一次方法调用时所占用的内存
  • 每个线程只能有一个活动栈帧，对应着当前正在执行的那个方法

线程上下文切换-> Thread Context Switch

由于一些原因导致CPU不在执行当前的线程，转而执行另一个线程的代码（即从使用CPU到不使用CPU称为上下文切换，切换时也会保存其上下文，如程序计数器、寄存器内容、堆栈指针等，再去加载另一个线程中的上下文，执行另一个线程）。

  • 线程的CPU时间片用完
  • 垃圾回收
  • 有更高优先级的线程需要运行
  • 线程自己调用了sleep、wait、synchronize、lock等方法

常用方法

方法名 static 功能说明 注意
start() 启动一个线程，在新的线程运行run方法中的内容
run() 新线程启动后会调用的方法 如果在够着Thread对象是传递了Runnable参数，则线程启动后会调用Runnable中的run方法，否则默认不执行任何操作。但可以创建Thread的子类对象，来覆盖默认行为（详见第一篇文章的两种创建线程方法的区别）
join() 等待线程运行结束
join(long n) 等待线程运行结束,最多等待n毫秒
getId() 获取线程长整型的id id唯一
getName() 获取线程名
setName(String) 修改线程名
getPriority() 获取线程优先级
setPriority(int) 修改线程优先级 java中规定线程优先级是1~10的整数，较大的优先级能提高该线程被CPU调度的几率
getState() 获取线程的状态 java中线程状态是用6个enum表示。分别为：New、Runnable、Blocked、Waiting、Timed_Waiting、Terminated
isInterrupted() 判断是否被打断 不会清除打断标记
isAlive() 线程是否存活（还没有运行完毕）
interrupt() 打断线程 如果被打断线程正在sleep、wait、join会导致被打断的线程抛出InterruptedException,并清除打断标记；如果打断的正在运行的线程，则会设置打断标记；park的线程被打断，也会设置打断标记
interrupted() static 判断当前线程是否被打断 会清除打断标记
currentThread() static 获取当前正在执行的线程
sleep(long n) static 让当前执行的线程休眠n毫秒。休息时间让出CPU的时间片给其它线程
yield() static 提示线程调度器让出当前线程对CPU的使用 主要是为了测试和调试
